# Файл весов модели

В этой папке должен находиться файл `yolov8n-seg.pt` — официальная
предобученная модель сегментации YOLOv8-nano от Ultralytics (лицензия
AGPL-3.0), обученная на датасете COCO (80 классов, включая `person`).

## Почему файла нет в репозитории

Файл весов — бинарный файл ~6-7 МБ, который распространяется Ultralytics
как вложение к GitHub Release, а не как обычный файл в git-репозитории.
Получить его можно двумя способами:

### Способ 1 (рекомендуется): автоматический скрипт

```
pip install -r requirements.txt
python scripts/fetch_weights.py
```

Скрипт скачает официальный файл и положит его сюда как `yolov8n-seg.pt`.
Это единственный момент, когда программе вообще нужен интернет — после
этого шага приложение полностью самодостаточно и офлайн.

### Способ 2: вручную

Скачайте `yolov8n-seg.pt` со страницы релизов Ultralytics:
https://github.com/ultralytics/assets/releases

и поместите файл в эту папку (`weights/yolov8n-seg.pt`) без переименования.

## Важное уточнение про "ночной" режим модели

В задании упоминалась модель, "оптимизированная под ИК/ночные режимы".
Официальной такой модели в открытом доступе не существует — стандартная
`yolov8n-seg.pt` обучена на обычных RGB-снимках. Вместо мифической
"ИК-модели" NeuroMovement использует честный и работающий подход:
кадр сначала проходит через CLAHE-улучшение контраста
(`neuromovement/vision/night_vision.py`), вытягивающее видимость силуэтов
в темноте, и только после этого попадает на вход YOLOv8-seg. Так силуэт
человека остаётся узнаваемым для сети даже в условиях слабой освещённости.

Если в будущем понадобится по-настоящему специализированная модель
(например, дообученная на кадрах именно с ваших камер ночного видения),
её нужно будет обучить отдельно (fine-tuning) и заменить ею файл в этой
папке — остальной код менять не придётся, так как SegmentationDetector
работает с любым checkpoint формата YOLOv8-seg.
