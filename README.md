# NeuroMovement

Автономное десктопное приложение для Windows: улучшает видимость в темноте
и ярко подсвечивает движущихся людей/объекты на видео с домашних камер.
Работает **полностью офлайн** — никаких серверов, никакой отправки видео
куда-либо. Вся обработка (улучшение контраста, распознавание, отрисовка
тревоги) происходит локально на вашем компьютере.

## Возможности

- **Источники видео**: RTSP-камера, захват целого монитора, захват
  конкретного открытого окна (Windows) — например, окна вашей программы
  видеонаблюдения.
- **Улучшение видимости в темноте**: CLAHE-препроцессинг вытягивает
  контраст и убирает "слепые зоны" при слабом освещении.
- **Распознавание силуэтов**: локальная модель YOLOv8-seg (сегментация)
  находит людей/объекты и строит их точный контур, попиксельно.
- **Тревожная визуализация**: полупрозрачная яркая заливка силуэта
  (красный / кислотно-зелёный / жёлтый на выбор), чёткий контур и
  пульсирующая рамка вокруг объекта — всё это появляется только когда
  есть реальное движение. В остальное время кадр абсолютно чистый: без
  засечек, прицелов, подсказок и текста поверх видео.
- **Автоматическое использование GPU**: если есть совместимая NVIDIA
  видеокарта — распознавание идёт на ней (CUDA); если нет — автоматически
  используется CPU.
- **Полностью офлайн**: приложение аппаратно блокирует любые исходящие
  сетевые запросы из Python-кода (см. раздел "Офлайн-режим" ниже).

## Требования

- Windows 10/11 (для захвата конкретного окна и готовой .exe-сборки).
  Захват RTSP и монитора работает и на Linux/macOS при запуске из исходников.
- Python 3.10 или новее — только для разработки/сборки. Готовый .exe не
  требует установленного Python.
- (Опционально) NVIDIA GPU с установленными драйверами — для ускорения
  через CUDA. Без GPU всё работает на CPU, просто медленнее.

## Быстрый старт (запуск из исходного кода)

```bash
# 1. Клонировать репозиторий
git clone https://github.com/WZCasper/NeuroMovement.git
cd NeuroMovement

# 2. Создать виртуальное окружение
python -m venv .venv
.venv\Scripts\activate        # Windows
# или: source .venv/bin/activate   # Linux/macOS

# 3. Установить зависимости
#    (первая установка скачает несколько ГБ — библиотека PyTorch по
#    умолчанию включает компоненты CUDA, даже если видеокарты нет;
#    это разовая стоимость установки, само приложение работает офлайн)
pip install -r requirements.txt

# 4. Скачать файл весов модели (единственный сетевой шаг во всём проекте)
python scripts/fetch_weights.py

# 5. Запустить
python main.py
```

## Использование

1. В правой панели выберите источник видео:
   - **RTSP-камера** — введите ссылку вида `rtsp://192.168.1.100:554/stream1`.
   - **Монитор** — выберите номер физического монитора для захвата.
   - **Окно приложения** (только Windows) — нажмите "Обновить список окон"
     и выберите окно вашей программы видеонаблюдения из списка. Окно
     должно быть видимо на экране (не свёрнуто).
2. Настройте визуализацию тревоги: цвет подсветки, пульсацию рамки,
   толщину обводки. Есть и дополнительные параметры: прозрачность
   заливки, порог уверенности распознавания, экономия ресурсов
   (не запускать распознавание, пока нет грубого движения в кадре).
3. Нажмите **"Запустить"**. Статус подключения и текущий FPS видны внизу
   окна. Настройки сохраняются автоматически в файл
   `neuromovement_settings.json` рядом с программой.

## Сборка портативного .exe

На Windows-компьютере с установленными зависимостями:

```bash
pip install -r requirements-dev.txt
python scripts/fetch_weights.py   # если ещё не скачан
python build.py
```

Готовая сборка появится в `dist/NeuroMovement/` — это самодостаточная
папка, её можно скопировать на любой Windows-компьютер и запускать
`NeuroMovement.exe` без установки Python.

## Автоматическая сборка через GitHub Actions

В `.github/workflows/build.yml` настроен пайплайн, который **автоматически
собирает Windows-версию на настоящей Windows-машине в облаке GitHub**
при создании нового Release в репозитории (Releases → Draft a new release).
Собранный `NeuroMovement-Windows.zip` прикрепляется к релизу автоматически.
Пайплайн также можно запустить вручную вкладке Actions → Build Windows
Release → Run workflow — удобно, чтобы проверить сборку, не публикуя релиз.

Модель весов при этом скачивается автоматически на самом раннере GitHub
(`scripts/fetch_weights.py` в первом шаге пайплайна), так что в самом
репозитории бинарный файл весов постоянно хранить не обязательно.

## Тесты

Проект покрыт автоматическими тестами (43 теста на момент публикации):
CLAHE-обработка, фильтр движения, отрисовка тревоги и логика пульсации,
захват видео (RTSP и монитор), сетевая блокировка, полный конвейер
обработки кадра и интеграционный тест интерфейса (запуск/остановка
мониторинга, получение и отображение кадров).

```bash
pip install -r requirements-dev.txt
python scripts/fetch_weights.py   # тест детектора требует torch/ultralytics
pytest tests/ -v
```

## Офлайн-режим: как это работает на самом деле

`neuromovement/network_guard.py` подменяет системный `socket.socket` сразу
при старте программы (`main.py`, первая же строка исполняемого кода) так,
что любая попытка установить исходящее соединение из Python-кода поднимает
исключение. Это перекрывает встроенную телеметрию/автообновления
библиотек `ultralytics` и `torch` независимо от того, в каком порядке они
импортированы. Дополнительно, при инициализации детектора приложение
явно отключает собственный механизм синхронизации Ultralytics
(`ultralytics.settings`) — двойная защита на случай, если на компьютере
уже стоит глобальный конфиг с включённой синхронизацией.

Захват RTSP-камеры при этом продолжает работать, потому что OpenCV
использует нативную библиотеку FFmpeg (код на C), которая обращается к
сети напрямую через ОС, в обход модуля `socket` интерпретатора Python.
Блокируется именно и только сетевая активность, инициированная из
Python-кода.

## Известные ограничения

- **Скорость на CPU**: без GPU распознавание одного кадра моделью
  yolov8n-seg занимает не доли секунды, а заметное время (порядок —
  единицы FPS на слабом CPU; на современном многоядерном CPU или тем
  более на GPU — заметно быстрее). Захват видео и его отображение при
  этом не подвисают: обработка кадра идёт в отдельном потоке, интерфейс
  остаётся отзывчивым в любом случае. Опция "Экономия ресурсов" в панели
  настроек существенно снижает нагрузку для статичных сцен, запуская
  распознавание только когда в кадре действительно что-то шевелится.
- **Захват окна**: реализован через определение координат окна (Windows
  API) и захват этой области экрана — окно должно быть видимо на экране
  (не свёрнуто и не полностью перекрыто другим окном).
- **"ИК-оптимизированная" модель**: как честно указано в
  `weights/README.md`, официальной специализированной под ИК-камеры
  модели YOLOv8-seg в открытом доступе не существует. Используется
  стандартная модель + программное усиление видимости через CLAHE.
